# dip 黑膠 Roguelike 模式 — 設計藍本

> 獨立於「收集＋對戰」之外的**第二個遊戲模式**。共用既有戰鬥引擎（`battle.html` 的氣勢／相生相剋／王牌被動），但**入口、牌組來源、勝負框架**完全不同。
> 卡牌數值、稀有度、三軸文化定位、戰鬥規則一律沿用 `CARD_GAME_DESIGN.md`（以下簡稱「核心藍本」），本文件只寫 roguelike 特有的層。
>
> 一句話定位：**「品味朝聖之旅」——開局寒酸三張盤，靠一趟旅程現組現養，輸了重來。**

---

## 1. 與現有對戰的區別（為何另開模式）

| | 現有對戰（核心藍本 §8） | Roguelike 模式（本文件） |
|---|---|---|
| 入口 | 帶卡冊最強 20 張 | 開局只挑 3 張種子 |
| 牌組來源 | 開戰前固定組好 | 一趟旅程中現撿、現淘、現升級 |
| 結構 | 單場 PvE | 連續多場（節點地圖）串成一趟 run |
| 失敗 | 單場輸贏 | **永久死亡**：run 結束、重來 |
| 養成 | 卡冊永久收藏 | run 內暫時牌組（洗掉）＋ run 外 meta 解鎖 |
| 共用 | — | **同一套戰鬥引擎**（氣勢／相剋／王牌），零重寫 |

> 兩模式共存、互不取代。對戰＝秀肌肉；roguelike＝考驗適應力。

## 2. 開局（核心設計）

### 2.1 選流派（＝選角色 / class）
三軸天生就是三個職業（對應核心藍本 §8.4 的屬性效果）。先選流派，決定 **run 級被動** 與堆疊方向：

| 流派 | 綁定軸 | run 級被動（playtest 初值，可調） | 玩法 |
|---|---|---|---|
| **守舊派〔守〕** | 經典 classic | 每場戰鬥起始 **HP +5**；經典屬性效果 +HP 加倍（+1→+2） | 高血線、耐久消耗 |
| **挖掘派〔挖〕** | 冷門 obscurity | 每回合**氣勢 +1** 起跳；相剋充能 +1（+3→+4） | 氣勢爆發、品味碾壓 |
| **異端派〔攻〕** | 入耳 accessibility | 入耳屬性效果加倍（對手氣勢 −1→−2）；對手充能更難 | aggro、壓制節奏 |

> 流派被動是「整趟 run 都生效」的軸向加成，不是單場 buff。這是 roguelike 的「角色差異化」靈魂。

### 2.2 選 3 張種子（從卡冊）
- 從玩家**卡冊實際擁有**的卡選 **3 張**當起始種子。
- **限制**：
  - **王牌（hall/pearl/heresy）不可當開局種子** —— 王牌要靠 run 中途撿到，否則沒有成長曲線。
  - 建議規定**至少含兩種屬性**，逼玩家想清楚開局方向（或與所選流派扣合）。
- 三張**不直接當牌組**（連起手 5 張都湊不滿）。種子＝牌組的「核心三張」。

### 2.3 補基礎雜牌 → 湊成寒酸起始牌組
- 系統自動配發一疊**基礎雜牌**（低數值、可被淘汰），與 3 張種子湊成約 **8～10 張** 的起始牌組。
- 這正是《殺戮尖塔》開局（角色核心牌 + 一堆 Strike/Defend）。開得寒酸 → 一路淘碟養肥 → 爽點來源。

> **為何讓玩家從卡冊選種子？** 讓永久收藏有意義（蒐集越多 → 開局選擇越多），又不像直接帶 20 張那樣變課金輾壓。

### 2.4 開局流程
1. 選**流派**（三選一，定 run 級被動）。
2. 從卡冊選 **3 張種子**（非王牌、建議扣合流派／至少兩屬性）。
3. 系統補基礎雜牌 → 約 8 張寒酸起始牌組。
4. 進節點地圖，開始旅程。

## 3. 旅程結構（節點地圖）

> P0 先**不做程序生成**：固定一條路線、5 場，驗證好不好玩；好玩了再做隨機地圖。

- 一趟 run ＝ 一條由**節點**串成的路線（爬塔／走地圖感）。
- 節點類型（P0 先做前三種）：
  - **⚔️ 對戰**：用既有戰鬥引擎打一場 PvE。輸＝run 結束。
  - **🎁 撿盤**：戰後三選一加入牌組（可含王牌、遺物）。
  - **🗑️ 淘碟**：免費淘汰牌組一張（roguelike 精煉核心）。
  - **🏪 商店**（P1）：用碎片換卡／遺物／淘碟。
  - **❓ 事件**（P1）：文字抉擇＋風險（「二手店翻到一張極冷門私盤，要不要？代價：佔牌組一格」）。
  - **👑 王（Boss）**（P1）：路線終點強敵，三圍／氣勢更兇。
- **區域主題**（P2）：用曲風／年代當分區（jazz 區、前衛區…），影響該區卡池與敵人傾向。

## 4. run 內成長 vs run 外永久

- **run 內（暫時，run 結束洗掉）**：撿到的卡、升級、遺物、當前牌組與 HP。
- **run 外（永久）**：
  - 卡冊收藏**不受 run 影響**（roguelike 不消耗永久資產）。
  - **meta-progression**：累積跑局解鎖新流派變體、新事件、新起始種子上限、新遺物進池。給「即使輸了也有長進」的動力。

---

## 5. 遺物 / 被動（roguelike 靈魂）

> 一場一場疊出 build。套主題很自然，以下為示意（數值待 playtest）：

| 遺物（示意） | 效果 |
|---|---|
| **黑膠唱針** | 每場戰鬥相剋命中多 +1 氣勢 |
| **絕版內襯** | 王牌打出時額外效果（依王牌類型） |
| **二手店常客卡** | 每進一個商店節點，首次淘碟免費 |
| **稀有度獵人徽章** | 撿盤節點多一個選項（三選一→四選一） |

> 遺物效果全部寫成 config，與 `BATTLE_CONFIG` 同風格，戰鬥開始時套用為臨時 modifier。

---

## 6. 戰鬥引擎複用（重要：幾乎不用改）

- 直接用核心藍本 §8 的氣勢制：HP、氣勢槽 0→6、相生連段、相剋壓制、屬性效果、王牌被動、品味碾壓、疲勞。
- **roguelike 模式的差異只是「傳給戰鬥引擎的初始狀態」不同**：
  - 牌組＝當前 run 牌組（不是固定 20 張）。
  - 起始 HP／氣勢＝套流派被動 + 遺物 modifier。
  - 敵人＝該節點配置（一般敵 / Boss）。
- 工程上：把 `battle.html` 的對戰邏輯抽成可被「帶入一組初始 config」啟動的函式，roguelike 模式呼叫它、戰後把結果（勝敗、剩血）回拋給 run 流程。

## 7. 存檔 / 資料結構（草案）

> run 是暫時的，但要能中途離開續玩，故存「進行中的 run」一份。

```
runs/{uid}/active            // 進行中的 run（只留一份；run 結束即刪或標記）
  {
    class: "obscurity",      // 流派
    seeds: [cardId, cardId, cardId],
    deck: [cardId, ...],     // 當前牌組（含雜牌＋撿到的）
    relics: [relicId, ...],
    hp: 24,                  // run 當前血（跨場帶著走？見待決定）
    node: 3,                 // 走到第幾個節點
    map: {...},              // 路線（P0 固定、P1 程序生成種子）
    seed: 123456,            // 隨機種子（可重現）
    startedAt, updatedAt
  }

users/{uid}/roguelikeMeta    // run 外永久進度
  { runsPlayed, bestDepth, unlocks: [...] }
```

- 卡牌數值仍走 Worker KV 目錄（核心藍本 §2），run 只存 `cardId` 與 run 特有狀態。
- 安全規則：`runs/{uid}/**`、`users/{uid}/roguelikeMeta` 比照核心藍本 §9，per-uid 自己讀寫。

## 8. 分階段路線圖

- **P0 最小可玩（先驗證好不好玩）**
  - 選流派（先實作 1～2 種）＋ 選 3 種子 ＋ 補雜牌湊起始牌組。
  - **固定路線 5 個節點**（對戰／撿盤／淘碟，不程序生成）。
  - 複用 `battle.html` 戰鬥；輸＝run 結束、重來。
  - 純前端、跑局狀態先放 localStorage（連 Firestore 都先不用）。
- **P1 旅程深化**：商店、事件節點、Boss、碎片經濟、王牌可在 run 中撿到。
- **P2 內容化**：程序生成地圖、區域主題（曲風／年代）、三流派補齊＋變體、遺物池擴充。
- **P3 永久化**：run 存 Firestore（跨裝置續玩）、meta-progression 解鎖、戰績／最深層數。
- **P4（評估）**：每日種子挑戰、排行榜。

> 每階段都能獨立上線、各自驗證。**先把 P0 在一個新分頁／新檔（如 `roguelike.html`）跑起來看好不好玩，再決定要不要往下做。**
