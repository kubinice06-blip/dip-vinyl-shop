# dip 黑膠 Roguelike 模式 — 設計藍本

> 獨立於「收集＋對戰」之外的**第二個遊戲模式**。共用既有戰鬥引擎（`battle.html` 的氣勢／相生相剋／王牌被動），但**入口、牌組來源、勝負框架**完全不同。
> 卡牌數值、稀有度、三軸文化定位、戰鬥規則一律沿用 `CARD_GAME_DESIGN.md`（以下簡稱「核心藍本」），本文件只寫 roguelike 特有的層。
>
> 一句話定位：**「無止盡的品味試煉」——開局寒酸三張盤，快打一場接一場、越打越強，撐到死、看你能走多深。**

### ⚡ 核心節奏：快打 ＋ 無限賽局（最重要的約束）
- **每場戰鬥超快**：平均只 **3～4 次出招（攻防交換）** 就分勝負。靠極低 HP／高傷害密度達成（見 §6），不是慢慢磨血。
- **整個 run 無限**：一場接一場、敵人**越打越強**，直到**輸（永久死亡）** 才結束。**沒有固定終點、沒有 Boss 收尾**——撐越久越強。
- **成績 ＝ 撐到第幾場 / 連勝數**（depth / streak），這是 meta 與排行的核心數字。
- 牌組在無限賽局中會**慢慢長大**（每場後可撿／淘），所以「開局選擇」與「途中養牌」都重要；但**每一場打起來都要快**。
- 目標體驗：**快節奏連打、越打越緊張、死了想再來一次。**

## 1. 與現有對戰的區別（為何另開模式）

| | 現有對戰（核心藍本 §8） | Roguelike 模式（本文件） |
|---|---|---|
| 入口 | 帶卡冊最強 20 張 | 開局只挑 3 張種子 |
| 牌組來源 | 開戰前固定組好 | 一趟旅程中現撿、現淘、現升級 |
| 結構 | 單場 PvE | **無限賽局**：快打一場接一場、越來越強，死了重來 |
| 單場長度 | HP 30、可拉長 | **超快**：HP 低、3～4 次出招分勝負 |
| 失敗 | 單場輸贏 | **永久死亡**：run 結束、重來 |
| 養成 | 卡冊永久收藏 | run 內暫時牌組（洗掉）＋ run 外 meta 解鎖 |
| 共用 | — | **同一套戰鬥引擎**（氣勢／相剋／王牌），零重寫 |

> 兩模式共存、互不取代。對戰＝秀肌肉；roguelike＝考驗適應力。

## 2. 開局（核心設計）

### 2.1 選流派（＝選角色 / class）
三軸天生就是三個職業（對應核心藍本 §8.4 的屬性效果）。先選流派，決定 **run 級被動** 與堆疊方向：

| 流派 | 綁定軸 | run 級被動（playtest 初值，可調） | 玩法 |
|---|---|---|---|
| **守舊派〔守〕** | 經典 classic | 每場戰鬥起始 **HP +5**；經典屬性效果 +HP 加倍（+1→+2） | 高血線、耐久消耗 |
| **挖掘派〔挖〕** | 冷門 obscurity | 每回合**氣勢 +1** 起跳；相剋充能 +1（+3→+4） | 氣勢爆發、品味輾壓 |
| **異端派〔攻〕** | 硬蕊 accessibility | 硬蕊屬性效果加倍（對手氣勢 −1→−2）；對手充能更難 | aggro、壓制節奏 |

> 流派被動是「整趟 run 都生效」的軸向加成，不是單場 buff。這是 roguelike 的「角色差異化」靈魂。
> **流派也給三圍基礎值**（見 §2.5）：守舊派偏高防、挖掘派偏高氣勢率、異端派偏高攻。

### 2.2 選 3 張種子（從卡冊）
- 從玩家**卡冊實際擁有**的卡選 **3 張**當起始種子。
- **限制**：
  - **王牌（hall/pearl/heresy）不可當開局種子** —— 王牌要靠 run 中途撿到，否則沒有成長曲線。
  - 建議規定**至少含兩種屬性**，逼玩家想清楚開局方向（或與所選流派扣合）。
- 三張**不直接當牌組**（連起手 5 張都湊不滿）。種子＝牌組的「核心三張」。

### 2.3 補基礎雜牌 → 湊成寒酸起始牌組
- 系統自動配發一疊**基礎雜牌**（低數值、可被淘汰），與 3 張種子湊成約 **8～10 張** 的起始牌組。
- 這正是《殺戮尖塔》開局（角色核心牌 + 一堆 Strike/Defend）。開得寒酸 → 一路淘碟養肥 → 爽點來源。

> **為何讓玩家從卡冊選種子？** 讓永久收藏有意義（蒐集越多 → 開局選擇越多），又不像直接帶 20 張那樣變課金輾壓。

### 2.4 開局流程
1. 選**流派**（三選一，定 run 級被動＋三圍基礎值）。
2. 從卡冊選 **3 張種子**（非王牌、建議扣合流派／至少兩屬性）。
3. 系統補基礎雜牌 → 約 8 張寒酸起始牌組。
4. 進入無限賽局，開始連打。

### 2.5 角色三圍（攻／防／氣勢率 — 暫名，待定）
> 這是**玩家與對手的角色級數值**，跟卡牌三軸（classic/obscurity/accessibility）**不同層**：卡牌三軸決定「出哪張、相生相剋」，角色三圍決定「打多痛、扛多少、爆多快」。名字先用暫名，做出來看到畫面再定（見 §9）。

| 暫名 | 作用 | 套在戰鬥引擎哪裡 |
|---|---|---|
| **攻擊** | 你打中時的**傷害加成** | 相剋命中／純比星／品味輾壓的傷害 ＋它（或 ×它） |
| **防禦** | 你受擊時的**減傷**（**最低仍扣 1**，不可完全免疫，免得拖垮快打） | 對手對你造成傷害時先扣防禦 |
| **氣勢率**（招牌維度） | **氣勢充能加成** → 充更快 → **更常放品味輾壓** | 各種氣勢 +N 時額外 ＋它 |

- **為何第三個綁氣勢**：氣勢槽＋品味輾壓是本作招牌，把第三圍綁它 → 三圍剛好是「打得痛／扛得住／爆發頻率」，且是**別的卡牌遊戲沒有的辨識度維度**。（備案：改做「節奏」＝每回合多抽 1／先攻，見 §9。）
- **三圍怎麼來**：流派給**基礎值** → 撿盤／發燒配件**疊加** → 升級（§4 聆聽經驗）拉**永久底值** → 敵人隨 depth **爬升**。
- **數值都寫 config**（`ROGUE_BATTLE_CONFIG`），加成用加法或乘法待 playtest 決定（快打偏好「小數值、加法」較好預測）。

## 3. 旅程結構（無限賽局：快打循環）

> **沒有終點。** 一場接一場直到死。核心是「快打一場 → 秒選獎勵 → 立刻下一場」的緊湊循環，敵人逐場變強。

**循環：**
```
[戰 N：快打 3~4 招] → 贏 →(秒選一個獎勵) → [戰 N+1：敵人更強] → … → 任一場輸 = run 結束、記 depth
                       輸 → run 結束（永久死亡），顯示撐到第幾場 / 連勝
```
- **每場之間只插一個「秒選獎勵」**（二選一，不堆瑣碎節點），選完馬上開下一場，維持連打手感。
- 秒選獎勵 = 隨機**擇一**呈現：
  - **🎁 撿盤**：二選一加入牌組（深層才可能出王牌）。
  - **🗑️ 淘碟**：免費淘汰牌組一張（精煉、提高核心牌出現率）。
- **難度爬升（每場遞增）**：敵人三圍／起始氣勢／AI 精明度隨 depth 提高；HP 只微升（避免破壞 3~4 招的快打）。爬升曲線寫 config。
- **場間是否回血**：見 §9 待決定（建議「贏了小回血、或完全不回」二選一實測）。

**之後才加（豐富循環，不破壞快打）：**
- **🏪 商店 / ❓ 事件**（P1）：偶爾穿插在獎勵節點位置（如每 5 場一次），用碎片換卡／發燒配件／淘碟、或文字抉擇＋風險。
- **里程碑強敵**（P1）：每 N 場一個特別強的敵人＋較好獎勵，當作節奏起伏（不是終點）。
- **區域主題**（P2）：每段 depth 換曲風／年代主題，影響卡池與敵人傾向。

## 4. run 內成長 vs run 外永久（含聆聽經驗／升級）

- **run 內（暫時，run 結束洗掉）**：撿到的卡、發燒配件、當前牌組、HP、本趟三圍加成。
- **run 外（永久）**：
  - 卡冊收藏**不受 run 影響**（roguelike 不消耗永久資產）。
  - **聆聽經驗（XP）＋ 等級**（見 §4.1）＝主要 meta-progression。
  - 隨等級解鎖：新流派變體、新事件、起始種子上限、新發燒配件進池、三圍永久底值。給「即使輸了也有長進」的動力。

### 4.0 兩層經驗（重要：局內 vs 結算，分開算）
> 仿《吸血鬼倖存者》：**戰鬥中**邊打邊升級拿強化（暫時、洗掉）；**結算**另給一條永久經驗。兩條 XP 互不相干。

| | 局內經驗（run 內） | 樂歷（結算，永久） |
|---|---|---|
| 何時拿 | **每場戰鬥勝利**累積 | **run 結束**依 depth／連勝結算 |
| 升級給 | **三選一強化**：三圍數據（攻/防/氣勢率）或**特殊能力**（最大HP、起手多抽、相剋多充氣勢、每回合回血、輾壓+傷、撿盤/淘碟） | **天賦點**（永久三圍底值）＋**抽專輯**（隨機進收藏） |
| 壽命 | run 結束**洗掉** | **永久**存 localStorage |
| 作用 | 這趟 build 越滾越強 | 帳號變強、收藏變多、開局選擇變多 |

- **局內**：`winFight` 給 `6+depth*2` XP，跨門檻 `8+(lv-1)*4` 即升級 → 跳出三選一面板（可連升多次）。強化存 `RUN.stats`／`RUN.mods`，整趟生效。
- **抽專輯 → 收藏 → 種子**：樂歷每升一級抽 1 張專輯進 `META.collection`；**開局種子從收藏挑**（新玩家送 8 張起手收藏）。收藏越大、開局陣容越多 → 形成長期養成迴圈，接上核心藍本的卡冊/gacha。

### 4.1 樂歷（結算經驗）與升級
> 主題：**「聽音樂的累積」＝經驗值。** 每趟 run 結束(無論輸贏)依表現給 XP，累積升級，玩家帳號**永久變強**。名稱定為「**樂歷**」。

- **何時給 XP**：每趟 run **結束時**結算（死亡或主動收手都算）。
- **XP 來源（建議公式，可調）**：`XP = depth × a ＋ 連勝最佳 × b ＋ 王牌擊敗數 × c ＋ 首次達成里程碑獎勵`。打越深給越多 → 鼓勵推進。
- **升級給什麼**（每級擇一或固定，待定）：
  - 三圍**永久底值 +1**（攻／防／氣勢率擇一，做成「天賦點」自選更有玩法）。
  - 解鎖新**流派／流派變體**、新**發燒配件**進池、新**起始種子格**（3→4 張）。
  - 純數字成長要設**遞減/上限**，避免高等級無腦輾壓 → 仍靠 run 內讀招。
- **這讓本作是 roguelite（帶永久成長）而非純 roguelike**——符合你要的「升級系統」，且和無限賽局的 depth 互相餵養（越強→撐越深→XP 越多）。
- **資料**：存 `users/{uid}/roguelikeMeta`（§7）：`xp`、`level`、已配點、解鎖清單。

### 5.0 發燒配件毀損／庫存出戰系統 — ✅ 已實裝（2026-07-13）
> 解決「發燒配件太強勢、拿到就永久」：黑膠器材會老，同時只能帶 3 件、且只有出戰的會磨損。

**開戰前選發燒配件（`renderRelicLoadout`）**：選完流派＋種子後，多一步從**持有中**（`META.relicsOwned`，靠打贏藏家累積；毀損會被移除）挑最多 3 件帶上場（可不帶）。沒持有任何發燒配件 → 自動跳過直接開打。「同流派再戰」`META.lastLoadout.relics` 沿用上次帶的（已毀損的自動跳過）。`META.relicsSeen` 仍是圖鑑「歷來拿過」的永久紀錄（點亮用）。

**庫存 ＋ 出戰欄（`RUN.relics` 庫存 / `RUN.equipped` 3 格）**
- 開戰前挑的＋藏家中途掉的發燒配件進**庫存**（無上限、不耗損）；出戰欄有空位自動裝上。藏家仍每 5 場掉發燒配件＝解鎖新發燒配件並可換裝。
- **只有出戰中的 fx 生效、也只有它們每場磨損**；庫存＝休息中的盤，收著不壞。
- `relicFx()` 只加總 `equipped` 且乘品相係數；`maxHp` 類（重量盤）改由 `relicMaxHp()` 動態算，品相下修/卸下會即時重算血上限（可能掉血上限，有戲）。
- 換裝在**戰鬥間**（勝利結算的「🎒 整備發燒配件」面板）自由做，戰鬥中鎖定。

**品相磨損（`WEAR` config，後台 `roguelike.wear` 可調／可關）— 2026-07-19 改版：唱片行式五階＋毀損即移除**
- 耗損值 0→100，五階（唱片行品相邏輯：**只有耐久 100 算全新**）：**M 全新**(耗損 0) ×1 → **NM 近全新**(1–39) ×1 → **VG 良品**(40–69) ×0.6 → **G 堪用**(70–99) ×0.3 → **F 毀損**(100) 0%。效果無條件進位、最低 1。耐久＝100−耗損。
- **毀損＝整件從收藏移除**（`applyWear` 結算時把該件從 `RUN.relics/equipped`＋`META.relicsOwned`＋耗損紀錄一併刪除），**只能再從傳說藏家手上拿回**（圖鑑 `relicsSeen` 仍點亮、顯示「已毀損 · 未持有」；等級 `relicLv` 保留，重拿回來直接沿用）。原「♻ 回收報廢件」機制隨之移除（沒有報廢件可回收了）。
- 每場每件出戰磨損 ＝ (基礎 `perFight`(6) + 該場 fx **真的觸發過**再 `usedBump`(5)) × **類別係數** × **tier係數** × **越強越脆** × **等級耐用**(`lvDura` 0.06/級，Lv.5＝×0.76)。
  - 類別（`RELIC_CAT`）：耗材 1.6（洗碟水最脆）/ 唱針機械 1.4 / 唱片 1.0 / 紙品 1.0 / 配件保護 0.6（最耐用）。五階語彙：耗材＝滿瓶/開瓶/半瓶/見底/空瓶、唱針＝全新/近全新/磨損/磨平/斷針。
  - tier2 ×0.6（更耐用）＋ fx 強度越高磨越快（越強越脆，以 Lv.1 基礎值算，升級不加磨）。
- **耗損跨趟保留**（`META.relicWear`，跟樂歷一起存本機＋雲端）：`applyWear` 每場勝利同步 `syncWearToMeta()`；開戰前 `carryRelic` 沿用舊耐久帶上場。掉落/禮物（藏家三選一新款、Lv.10 開局件）＝拿到「新的一件」→ 跨趟耗損歸零。
- **迴光一轉**：耗損 ≥ `swanAt`(82) 的搖搖欲墜期，該發燒配件 fx **翻倍**（黑膠最後一轉）——之後撐不住就是真的毀損消失，張力拉滿。洗碟水的每勝保養在毀損判定**前**結算，最後一口氣有機會救回來。

**⬆ 配件等級（Lv.1~5，`META.relicLv`；2026-07-19 新增）**
- 每件配件有等級：fx 數值＝`fx`(Lv.1 起始值，整體壓低) 與 `fx5`(Lv.5 滿級值) 線性內插（`fxAt`）；開關型效果（防塵套/集點卡/防靜電內套）各級同值，吃通用的 `lvDura` 耐用加成。神秘見本盤的免死回魂 HP 也隨等級走（revive 值＝回魂 HP，4→10）。
- 升級管道兩條：① **藏家三選一撞同款**＝免費升一級＋整新到全新（掉落池含「已持有未滿級」的同款，卡片標「⬆ 升級 Lv.n→n+1 · 整新」；滿級 Lv.5 的退出掉落池）；② **花現金升級**（`openRelicCare` 內，`upCost`[60/100/150/220]、tier2 ×`upT2`(1.5)，不附整新）——長期存錢目標。
- 等級跨趟、跨毀損保留（帳號合併逐件取較高）；desc 用 `{v}` 佔位帶入當前等級數值（`relicDescOf`）。

**💰 現金與保養（`CASH` config，後台 `roguelike.cash` 可調）— 趟中沒有免費保養，修哪件要錙銖必較**
- 收入：每贏一場 `base`(4)＋⌊depth×`perDepth`(0.5)⌋；擊敗傳說藏家再 +`bossBonus`(12)；趟結算破最深紀錄 +`recordBonus`(25)。現金跨趟累積（`META.cash`，帳號合併取較高者；耗損逐件取較低者）。
- 保養**只能在趟與趟之間**（主畫面/死亡結算的「發燒配件」圖鑑、開戰前選單）：點配件開 `openRelicCare` 視窗＝簡介＋耐久條＋兩檔保養——🧽 小保養 +`tuneStep`(25) 耐久、✨ 整新到全新；工錢＝每點耐久 tier1 `repairT1`($0.6) / tier2 `repairT2`($0.9)，經濟上一趟收入約只夠修四到六成耗損 → 得分配資源；沒修到 0 就是整件消失，保養的優先序更重要了。
- 趟中僅剩兩個管道：洗碟水出戰每勝 −`cleanerHeal`(8)、藏家獎勵「整新一件」（同步跨趟）。

**UI**：頂欄常駐 💰 存款；勝利畫面顯示本場收入；死亡結算顯示本趟收入＋「🔧 保養／升級發燒配件」入口；戰鬥列（相生相剋那列）**左側**橫向發燒配件欄＝像素圖 + 5 格耗損條（綠 M→黃綠 NM→橘 VG→橘紅 G→灰 F），輕點看效果／品相／耐久／Lv；勝利結算下方「🎒 整備發燒配件」開換裝面板（無保養鍵）＋耗損播報（毀損移除/降階）。各列表（開戰前選單/整備/圖鑑/保養）都顯示 Lv 標籤。

**像素圖隨等級進化（`PIX_ICONS_LV`；2026-07-19）**：每件配件 5 張專屬 12×12 像素圖，Lv.1 陽春／磨舊 → Lv.2 標準 → Lv.3 進階材質 → Lv.4 鍍金精品 → Lv.5 傳說（金件＋星光，外圈加淡金光暈 `.pixlv5` 脈動、`prefers-reduced-motion` 關閉）。`relicIconHTML(r,size,lvOverride)` 依當前等級取 `PIX_ICONS_LV[icon][lv-1]`；後台自訂件（無此表）退回 `PIX_ICONS`、再無則 emoji。藏家三選一的「升級」卡片圖示先預覽下一級（`lvOverride=lv+1`）。全站顯示點沿用同函式自動生效。

## 6. 戰鬥引擎複用（重要：幾乎不用改）

- 直接用核心藍本 §8 的氣勢制：HP、氣勢槽 0→6、相生連段、相剋壓制、屬性效果、王牌被動、品味輾壓、疲勞。
- **roguelike 模式的差異只是「傳給戰鬥引擎的初始狀態」不同**：
  - 牌組＝當前 run 牌組（不是固定 20 張；起手約 8 張）。
  - 起始 HP／氣勢＝套流派被動 + 發燒配件 modifier。
  - 敵人＝該節點配置（一般敵 / Boss）。
- **為「3~4 招分勝負」調短單場**（roguelike 專用 override，不動現有對戰）：
  - HP 大幅調低：**約 10～14**。算術上：相剋命中扣 1~7、品味輾壓 −6，HP 12 時約 2~4 次有效命中就見血光 → 對齊「3~4 招」目標。
  - 起手約 5 張即可（牌組本來就小），氣勢槽門檻可略降讓爆發更快來。
  - 敵人隨 depth 變強主要靠**三圍／氣勢／AI**，HP 只微升，**不要靠加大 HP 來提難度**（會拖慢、毀掉快打）。
  - 這些放 roguelike 自己的 config（如 `ROGUE_BATTLE_CONFIG`），原 `BATTLE_CONFIG` 不變。
- 工程上：把 `battle.html` 的對戰邏輯抽成可被「帶入一組初始 config」啟動的函式，roguelike 模式呼叫它、戰後把結果（勝敗、剩血）回拋給 run 流程。

## 7. 存檔 / 資料結構（草案）

> run 是暫時的，但要能中途離開續玩，故存「進行中的 run」一份。

```
runs/{uid}/active            // 進行中的 run（只留一份；run 結束即刪或標記）
  {
    class: "obscurity",      // 流派
    seeds: [cardId, cardId, cardId],
    deck: [cardId, ...],     // 當前牌組（含雜牌＋撿到的）
    relics: [relicId, ...],
    stats: { atk: 3, def: 2, qiRate: 1 },  // 角色三圍（流派基礎＋發燒配件＋永久底值，暫名）
    hp: 11,                  // run 當前血（無限賽局跨場帶著走；場間是否回血見 §9）
    depth: 7,                // 已打贏第幾場（無終點，越深越強）= 成績
    streak: 7,               // 連勝數
    seed: 123456,            // 隨機種子（可重現獎勵／敵人序列）
    startedAt, updatedAt
  }

users/{uid}/roguelikeMeta    // run 外永久進度（聆聽經驗／升級）
  {
    xp: 1240, level: 8,      // 聆聽經驗與等級（每趟 run 結束累加，§4.1）
    talents: { atk: 2, def: 1, qiRate: 0 },  // 升級配的三圍永久底值
    runsPlayed, bestDepth, bestStreak,
    unlocks: [...]           // 解鎖的流派／發燒配件／種子格等
  }
```

- 卡牌數值仍走 Worker KV 目錄（核心藍本 §2），run 只存 `cardId` 與 run 特有狀態。
- 安全規則：`runs/{uid}/**`、`users/{uid}/roguelikeMeta` 比照核心藍本 §9，per-uid 自己讀寫。

## 8. 分階段路線圖

- **P0 最小可玩（先驗證好不好玩）**
  - 選流派（先實作 1～2 種）＋ 選 3 種子 ＋ 補雜牌湊起始牌組（~8 張）。
  - **無限賽局循環**：快打一場 → 秒選獎勵（撿盤／淘碟二選一）→ 立刻下一場，敵人逐場變強。
  - 複用 `battle.html` 戰鬥，套 roguelike 縮短 config（**HP ~12、3~4 招分勝負**）＋**角色三圍**（攻/防/氣勢率，暫名）。
  - 輸＝run 結束、顯示 **depth／連勝**、重來。**目標：每場數十秒、死了想再來。**
  - **聆聽經驗／升級**：run 結束給 XP、簡單升級（先給三圍永久 +1）；先和跑局一起放 localStorage。
  - 純前端、狀態先放 localStorage（連 Firestore 都先不用）。
- **P1 旅程深化**：商店、事件節點、Boss、碎片經濟、王牌可在 run 中撿到。
- **P2 內容化**：程序生成地圖、區域主題（曲風／年代）、三流派補齊＋變體、發燒配件池擴充。
- **P3 永久化**：run 存 Firestore（跨裝置續玩）、meta-progression 解鎖、戰績／最深層數。
- **P4（評估）**：每日種子挑戰、排行榜。

> 每階段都能獨立上線、各自驗證。**先把 P0 在一個新分頁／新檔（如 `roguelike.html`）跑起來看好不好玩，再決定要不要往下做。**
